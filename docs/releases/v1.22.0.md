# v1.22.0 — 2026-07-07

## Breaking change: MCP surface — no `dysflow_` prefix anywhere

Thirteen legacy `dysflow_*` MCP tools are renamed to their canonical
(unprefixed) names. This is a continuation of the Opción A reference
commit (`58405eb2`, v1.21.0) which renamed 7 form/lint tools. v1.22.0
finishes the rename for the remaining 13 tools; **no back-compat
alias is added** — the canonical name is the only name advertised on
the MCP `tools/list` projection.

Any consumer (human or AI agent) that calls a `dysflow_*` MCP tool
must migrate to the canonical unprefixed name. The previous version
(v1.21.0) removed the alias for 7 form and lint tools; v1.22.0
removes the alias for the remaining 13.

## Notes

- The canonical `run_vba`, `list_access_operations`, and
  `cleanup_access_operation` were already registered by `alias-tools.ts`
  (preexisting aliases). The bespoke registration that lived
  alongside each was removed; the alias handler is now the sole source.
- The other ten canonical names are new and have no pre-existing
  alias registration; their handlers were moved from
  `src/adapters/mcp/tools.ts` exactly once.
- The two final renames (`get_capabilities`, `resolve_project`) were
  originally left behind because they were intentional canonical
  names registered in bespoke factory files
  (`get-capabilities-tool.ts` and `dysflow-resolve-project-tool.ts`),
  not as legacy aliases of an already-canonical tool. The
  `dysflow-resolve-project-tool.ts` file is renamed to
  `resolve-project-tool.ts` to match the canonical name.
- The `MCP_TOOL_CONTRACTS.dysflow_*` entries were renamed or removed
  in lockstep. Tools whose contract was previously in
  `modernContracts` (the 8 new canonical names and the pre-existing
  `run_vba`) now read from `modernContracts.<canonical>` (or
  `aliasContracts.<canonical>` for the 3 pre-existing aliases).
- Per-tool schemas: `MODULE_LIST`, `MODULE_GET`, and the modern
  canonical surface now use `mcpSchemaFor(<canonical>)`. The legacy
  schemas `MODULE_LIST` / `MODULE_GET` (which still existed as the
  bespoke `dysflow_*` schemas) were folded into the canonical schemas.
- `MCP_TOOL_SCHEMAS` and `TOOL_DESCRIPTIONS` carry the canonical
  names; the legacy `MODERN_TOOL_NAMES` array no longer mentions any
  `dysflow_*` string.

## Mechanical enforcement

A new regression test in
`test/adapters/mcp/advertised-tool-count.test.ts` asserts that no
advertised MCP tool name starts with `dysflow_`. This is the contract
for all future releases — the Opción A directive ("no legacy ni non
legacy, todos han de ser iguales") is now mechanically enforced.
